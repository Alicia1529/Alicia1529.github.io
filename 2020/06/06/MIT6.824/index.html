MIT 6.824 Distributed Systems lecture 1

Author: Alicia Luo Date: June 6, 2020  19:07:22 Category: Distributed Systems Views:

Reasons for it

  • parallelism
  • fault-tolerance
  • physical distance
  • security / isolation

Challenges

  • concurrency
  • partial failure
  • performance

Infrastructure

achieve Abstractions

  • storage
  • communications
  • computation

Implimentations

  • RPC (roles of procedure calls)
  • threads
  • concurrency control: locks

Performance

  • scalability - 2 X computers -> 2 X throughput
    • distribute users to different web servers

Fault-tolerance:

  • hide failtures and handle them
  • availability
  • recoverability: non-volatile storage (expensive to update), replication

Consistency

eg. what does put/get mean when there is more than one copy
strongly consistent: expensive implimentation
weakly consistent: does not guarantee the lastest written value
put copies far enough to preserve recoverability, but take a while to update
